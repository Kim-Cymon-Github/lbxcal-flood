# cal-flood 작업 규칙

설계/scope는 [Cal 모듈 플러그인 설계 문서](doc/Cal_Module_Plugin_Design.md) 참조 (구조·범위·소스 of truth·HKMC 변종 전부).

## 파일 인코딩

- VS에서 컴파일되는 소스 (`*.c`, `*.cpp`, `*.h`) → **UTF-8 BOM**.
- 그 외 파일 → **UTF-8 (no BOM)**.

## 줄바꿈

- 기본 **LF**.
- 예외: `*.bat` 등 CRLF가 강제되는 파일.

## 들여쓰기

- 기본 **Space 4개**.
- 예외: `Makefile` 등 Tab이 문법상 필요한 파일.

## 중괄호 스타일

- **제어문** (`if`/`else`/`for`/`while`/`switch`) → **K&R** (여는 괄호 같은 줄):
  
  ```c
  if (cond) {
      ...
  } else {
      ...
  }
  ```
- **함수/멤버함수 정의** → **Allman** (여는 괄호 다음 줄):
  
  ```c
  void func(int x)
  {
      ...
  }
  ```
- `{}`는 **어떤 경우에도 생략 불가** (단일 문장이라도).
- 짧은 return 등은 **한 줄 표기 허용**:
  
  ```c
  if (!ptr) { return NULL; }
  ```

## 주석

- `*.h` (외부 개발자용): **영어**, Doxygen 서식. 공개 API의 계약만.
- `*.c`, `*.cpp` (내부 개발자용): **한국어**, Doxygen 서식. 설계 의도·트레이드오프·민감한 구현 디테일까지 적어도 됨.

## 대화 규칙

- 대화는 합쇼체(~습니다, ~입니다, **극존칭 금지**), 문서는 해라체 (~이다 ~한다)
